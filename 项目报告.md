# 多智能体经济体可视化实验平台项目报告

**项目名称：AI Economist Visual Lab**
**作者：刘沛毓**

## 一、项目概述

本项目设计并实现了一个面向课程展示的多智能体经济体可视化实验平台。项目以“政策如何影响经济体中的公平、效率与资源可持续性”为核心问题，通过网页端二维地图模拟多个智能体在资源环境中的移动、采集、交易、建造和种树行为，并进一步比较不同政策模板下社会福利、平等度、生产力和生态指数的变化。

传统课堂中，公平与效率、税收与再分配、个体行为与宏观结果之间的关系往往比较抽象。如果只通过文字或静态图表解释，学习者很难直观看到政策参数改变后，个体行为如何发生变化，以及这些行为如何逐步累积为宏观层面的财富分布和社会福利结果。因此，本项目希望把抽象的政策机制转化为动态仿真过程。

项目参考了 AI Economist 一类研究中“多智能体环境 + 政策调节 + 社会目标函数”的基本思想，系统支持自由市场、AI Economist、累进税制、公平优先、效率优先等固定政策模板，同时加入一个简化版 Q-learning 规划器，用于展示政策可以根据环境反馈进行动态调整。

## 二、项目目标

本项目的主要目标包括三个方面。

第一，构建一个可视化的多智能体经济环境。系统中的 Agent 不再是静态数据点，而是在地图上持续行动的个体。每个 Agent 会根据自身资源、角色分工和环境状态，通过行动层 Q-learning 在采集、采矿、交易、建造、种树、移动和休整等行为中做选择，从而形成一个小型经济体的运行过程。

第二，实现不同政策模板下的对比实验。系统内置多种政策方案，包括自由市场、AI Economist、累进税制、公平优先和效率优先。不同政策在税率、再分配强度、劳动成本、建造收益、交易收益、公平权重和生态加成方面存在差异。用户可以切换政策并重新运行仿真，观察政策参数变化如何影响财富分布、经济产出和生态恢复。

第三，引入强化学习机制，展示动态决策思路。项目中一方面用行动层 Q-learning 控制 Agent 的微观行为，另一方面用年度政策 Q-learning 规划器决定下一年采用哪一种政策模板。

## 三、系统功能设计

平台整体采用网页应用形式，主要由实验配置区、经济体地图区、策略读数区和行为分析区组成。

实验配置区用于设置仿真参数，包括目标函数、政策模板、Agent 数量、资源密度、仿真年份和每年步数。系统还提供了预设场景，例如公平优先、效率优先和 RL 调节等，方便用户快速进入典型实验。此外，系统支持根据文本描述自动推断实验配置，例如用户输入“想做一个兼顾公平、效率和生态恢复的强化学习实验”，系统可以自动调整目标函数、政策类型、Agent 数量和资源密度等参数。

经济体地图区是项目的核心展示部分。地图上包含 Agent、树木、矿石、市场和房屋等元素。Agent 会在地图上移动，并根据行动价值表选择当前行为。用户可以播放、暂停或逐步查看仿真过程，也可以点击某个 Agent 查看其财富、资源库存、房屋数量和当前行动状态。通过这一模块，用户能够直观看到微观行为如何随着时间演化。

策略读数区用于展示当前政策下的宏观指标，包括社会福利、平等度、生产力、生态指数、平均财富和市场热度等。系统会展示 Agent 行动学习的训练轮数、状态数量、平均回报和动作分布；若使用 Q-learning 规划器，还会展示政策规划器的训练参数以及每一年选择的政策模板。

行为分析区主要用于结果解释和策略对比。系统会展示当前策略与基准策略的对照结果，包括福利、平等度、生产力和生态指标的差异。同时，平台还提供财富分布图、行为统计、事件流和年度趋势信息。用户可以将实验结果导出为 CSV 或 JSON 文件，便于后续整理报告或进一步分析。

## 四、技术架构与实现方式

本项目采用 React、TypeScript 和 Vite 开发。React 负责页面组件和交互状态管理，TypeScript 用于约束数据结构和减少类型错误，Vite 负责本地开发服务和项目打包。样式部分主要使用普通 CSS，并结合 Grid 和 Flexbox 实现布局，使页面能够同时展示配置面板、地图、指标卡片和分析图表。

项目的核心仿真逻辑集中在 `src/simulation.ts` 文件中。该文件定义了政策、智能体、资源节点、房屋节点、仿真快照、强化学习训练结果等数据结构，并实现了仿真推进、指标计算、税收再分配、强化学习训练和实验结果生成等主要逻辑。

页面交互和可视化部分集中在 `src/App.tsx` 文件中。该文件负责渲染实验配置表单、经济体地图、Agent 状态卡片、指标图表、策略解释面板和数据导出按钮。通过将仿真逻辑和界面逻辑分离，项目结构较为清晰，后续也更容易扩展新的政策、角色或资源类型。

项目可以通过如下命令运行：

```bash
npm install
npm run dev
```

如果需要构建生产版本，可以运行：

```bash
npm run build
npm run preview
```

## 五、多智能体经济环境设计

系统中的 Agent 分为采集者、建造者、交易者和林业者等角色。不同角色具有不同的行为倾向。例如，采集者更容易寻找树木并获取木材，建造者在拥有足够木材和石材时更倾向于建造房屋，交易者会向市场移动并出售资源，林业者则更倾向于种树和恢复生态。

地图中的资源主要包括树木和矿石。树木具有成熟和幼苗状态，成熟树木可以被砍伐，砍伐后会进入恢复过程；种树行为可以增加新的树木节点，并提高生态指数。矿石可以被采集并转化为 Agent 的石材库存。房屋则由建造者消耗木材和石材建造，建造后会带来财富收益和生产力提升。

每一步仿真中，系统会把 Agent 的角色、位置、资源库存、能量、财富水平、附近资源、市场位置、生态状态和当前政策编码成离散状态，再由行动层 Q-learning 选择具体动作。行动完成后，Agent 的财富、资源、能量和位置会发生变化，系统也会同步更新生产力、市场热度和生态状态。每一年结束时，系统会根据政策税率对 Agent 的年度收入征税，并按照再分配强度将财政收入部分返还给所有 Agent，低于平均财富的 Agent 会获得更强的支持。

## 六、政策模板设计

系统内置多种政策模板，每种政策代表不同的治理思路。

自由市场政策税率较低，再分配强度较弱，劳动成本较低，交易和建造收益较高。该策略通常能够刺激市场活跃和生产力提升，但财富差距也更容易扩大。

AI Economist 模板是一种折中型政策，在税收、再分配、生产激励和生态恢复之间保持相对均衡。它适合作为主实验或基准实验，用于展示政策在公平、效率和可持续性之间的综合权衡。

累进税制具有更强的再分配效果，能够在一定程度上控制财富分化。与自由市场相比，它更强调公平性，但可能会牺牲部分生产激励。

公平优先政策进一步提高税率和再分配强度，同时给予更高的公平权重和生态修复加成。该策略适合展示强再分配政策下财富差距收敛的过程，但在部分情况下可能导致生产力增长放缓。

效率优先政策则强调产出、交易和建造激励。它能够提高市场活跃度和经济产出，但也更容易产生财富分化和生态压力。

通过这些政策模板，系统能够展示不同政策目标之间的张力：追求效率可能带来更高产出，但不一定带来更高公平；强调公平可以压缩财富差距，但可能降低部分经济激励；兼顾生态则需要在短期收益和长期可持续性之间进行平衡。

## 七、Q-learning 设计

项目中的强化学习部分采用简化版 Q-learning，并分成两个层次。

第一层是 Agent 行动学习。每个 Agent 的状态由角色、库存、能量、财富水平、附近资源、市场位置、生态状态和当前政策组成，动作包括采集、采石、种树、建造、交易、移动和休整。奖励函数会综合考虑财富变化、库存变化、建房、种树、交易、生产力变化、生态变化和体力消耗。这样一来，Agent 的行为不再只是固定规则，而是可以根据训练结果形成行动偏好。

第二层是年度政策规划。政策规划器的动作不是 Agent 的微观移动，而是下一年选择哪一种政策模板。这样的设计使强化学习模块更加符合“政策调节”的含义，也便于解释其经济学含义。

系统将当前经济体状态编码为若干离散维度，包括目标函数、年份阶段、平等度水平、生产力水平和生态水平。平等度、生产力和生态指标会被划分为低、中、高三个状态区间，年份则被划分为早期、中期和后期。基于这些状态，Q-learning 规划器从候选政策中选择一个动作，并在运行一年的仿真后根据结果计算奖励。

奖励函数会根据用户选择的目标函数发生变化。如果目标是公平优先，奖励函数更重视平等度和社会福利；如果目标是效率优先，奖励函数更重视生产力、市场热度和社会福利；如果目标是综合平衡，奖励函数则同时考虑社会福利、生态、平等度和生产力。

训练过程中，系统使用探索率逐步下降的方式进行策略搜索。训练完成后，规划器会在正式仿真中根据学到的 Q 值逐年选择政策模板，并记录每一年的状态、候选政策价值和实际选择结果。虽然该模块是轻量化实现，但它已经能够展示强化学习如何根据环境反馈调整政策选择。

## 八、实验观察与结果分析

从项目运行效果来看，不同政策模板会带来较明显的系统差异。

在自由市场和效率优先策略下，Agent 的交易与建造行为通常更加活跃，生产力和市场热度更容易上升。这说明较低税率和较高交易、建造激励能够增强个体经济活动。但与此同时，财富会更快向部分高收益 Agent 集中，平等度可能下降，体现出高产出与高分化并存的现象。

在累进税制和公平优先策略下，系统通过更高税率和更强再分配机制压缩财富差距，平等度通常更容易维持在较高水平。但由于劳动成本、建造收益和交易收益受到政策参数影响，部分情况下生产力提升速度可能弱于效率优先策略。这一结果体现了公平与效率之间的常见权衡。

AI Economist 模板整体表现更为均衡。它不会像自由市场和效率优先那样过度强调产出，也不会像公平优先那样强力再分配，而是在福利、平等度、生产力和生态之间形成较为稳定的折中结果。因此，该策略适合用于说明综合型政策如何在多个目标之间寻找平衡。

Q-learning 规划器的主要价值在于展示动态政策调节。固定政策在整个仿真周期中保持不变，而 Q-learning 规划器会根据当前状态选择不同政策，使政策选择具有反馈机制。虽然它不是复杂的深度强化学习模型，但能够帮助理解“根据环境状态调整政策”这一思想。

## 九、项目亮点

本项目的第一个亮点是将抽象的经济政策问题转化为动态可视化系统。用户不仅能看到最终指标，还能看到 Agent 在地图上的具体行为，从而把微观行为和宏观结果联系起来。

第二个亮点是同时支持固定政策对比和强化学习策略调节。固定政策适合展示不同治理思路的差异，强化学习模块则进一步展示政策可以根据反馈动态调整，使项目具有更强的算法色彩。

第三个亮点是系统具有较完整的交互流程。用户可以配置实验、运行仿真、查看回放、比较策略、解释结果，并导出数据。这使项目不仅是一个静态展示页面，也可以作为一个小型实验平台继续扩展。

第四个亮点是项目结构清晰。仿真逻辑、政策设计、强化学习训练和前端展示之间具有较好的模块划分，便于后续继续添加新政策、新角色、新资源类型或更复杂的学习算法。

## 十、不足与改进方向

本项目目前仍然是一个课程展示型平台，而不是严格意义上的经济学结构模型或高精度仿真实验系统。首先，Agent 已经加入了行动层 Q-learning，但状态空间和奖励函数仍然是手工设计的，距离真实个体决策还有差距。后续可以进一步加入更丰富的历史记忆、风险偏好和个体差异。

其次，Q-learning 规划器的状态空间和动作空间较小。当前动作是离散政策模板，不能直接学习连续税率或连续再分配比例。后续可以将政策参数连续化，使用更复杂的强化学习算法进行优化。

第三，当前资源和产业链较为简单，主要包括树木、矿石、市场和房屋。未来可以加入农田、工坊、物流节点、消费品市场等元素，使经济系统更接近真实经济链条。

第四，当前实验主要展示单次运行结果。由于仿真中存在随机种子和随机扰动，后续可以加入批量实验功能，对同一政策运行多次并输出均值、方差和置信区间，从而增强结果的稳健性和分析可信度。
